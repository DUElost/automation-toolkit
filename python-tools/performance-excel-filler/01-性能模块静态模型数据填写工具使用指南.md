# Performance Excel Filler 使用指南
> **工具开发与维护者：dai.lv@tinno.com**

## 重要提示

### 首次使用必须验证数据准确性

> **警告：工具执行依赖预设的代码逻辑进行数据回填，结果不一定 100% 正确！**

首次使用时，请务必执行以下验证步骤：

1. **运行工具生成结果文件**
2. **与手动填充的数据进行逐项比对**
3. **确认数据准确性后再进行后续使用**
4. **如发现任何偏差，请立即联系工具开发者**

### 免责声明

本工具旨在提高数据填充效率，但自动化处理存在以下风险：

- 数据匹配可能因格式变化而失效
- 特殊情况可能未被代码逻辑覆盖
- 工具错误可能导致数据填入错误位置

**使用者有责任验证生成数据的准确性。如发现问题，请及时向工具开发者反馈。**

## 前置要求

### Python 环境

- Python 版本：**3.8 或更高版本**
- 检查版本命令：
  ```bash
  python --version
  ```

### 安装依赖

首次使用前，需安装项目依赖：

```bash
pip install -r requirements.txt
```

依赖包列表：
- pandas >= 2.0.0
- openpyxl >= 3.1.0
- xlrd >= 2.0.0
- python-dotenv >= 1.0.0

## 快速启动

双击运行 `Start.bat` 即可启动工具。

## 使用流程（交互式批量模式）

执行命令：
```bash
python main.py -b -i
```

### 步骤 1：选择设备类型

```
请选择设备类型:
  1. 测试机
  2. 竞品机
  3. 全选
  0. 跳过（不进行数据回填）

请输入选项:
```

| 选项 | 说明 |
|------|------|
| 1 | 仅处理测试机数据 |
| 2 | 仅处理竞品机数据 |
| 3 | 同时处理测试机和竞品机数据 |
| 0 | 跳过此步骤（不处理） |

### 步骤 2：选择数据类型

```
请选择数据类型 (可多选，用逗号分隔):
  1. 动效丢帧
  2. 滑动丢帧
  3. 全选
  0. 跳过（不进行数据回填）

请输入选项:
```

| 选项 | 说明 |
|------|------|
| 1 | 仅处理动效丢帧数据 |
| 2 | 仅处理滑动丢帧数据 |
| 3 | 同时处理两种数据类型 |
| 0 | 跳过此步骤（不处理） |

### 步骤 3：输入源目录路径

根据之前的选择，工具会依次提示输入对应的源目录路径：

**测试机动效丢帧源目录（默认: W:\KN5项目\静态模型\KN5\动效丢帧）:**
- 直接回车使用默认路径
- 或输入实际路径后回车

**测试机滑动丢帧源目录（默认: W:\KN5项目\静态模型\KN5\滑动丢帧）:**
- 直接回车使用默认路径
- 或输入实际路径后回车

**竞品机源目录（如果选择了竞品机）:**
- 同样方式输入动效丢帧和滑动丢帧源目录

### 步骤 4：选择用例等级

```
请选择用例等级筛选:
  1. L1
  2. L2
  3. L3
  0. 不筛选（全部）

请输入选项:
```

| 选项 | 说明 |
|------|------|
| 1 | 仅处理 L1 级别用例 |
| 2 | 仅处理 L2 级别用例 |
| 3 | 仅处理 L3 级别用例 |
| 0 | 处理所有级别用例 |

### 步骤 5：确认执行

工具显示执行摘要：

```
============================================================
Excel数据转移工具 - 批量模式
============================================================
设备类型: 测试机, 竞品机
数据类型: 动效丢帧, 滑动丢帧
测试机动效源目录: W:\KN5项目\静态模型\KN5\动效丢帧
测试机滑动源目录: W:\KN5项目\静态模型\KN5\滑动丢帧
竞品机动效源目录: W:\三星A07\动效丢帧
竞品机滑动源目录: W:\三星A07\滑动丢帧
目标文件: 从模板生成
用例等级筛选: L1
使用模板: 是
============================================================
```

确认无误后，按回车开始执行。

### 常见数据差异原因

| 原因 | 说明 |
|------|------|
| 文件夹命名格式变化 | 工具依赖特定的文件夹命名规则 |
| Excel 列名变化 | 模板列名变化可能导致匹配失败 |
| 新增/删除用例 | 用例数量变化可能影响索引匹配 |
| 数据源格式变化 | 源数据 Excel 格式变化可能导致读取失败 |


## 故障排除

### 问题：找不到源目录

```
源目录不存在: W:\xxx\xxx
```

**解决方法：**
- 检查路径是否正确
- 确认网络驱动器（如 W: 盘）是否已连接

### 问题：模板文件不存在

```
模板文件不存在: xxx\backup\静态模型 - V2.0 - KN8.xlsx
```

**解决方法：**
- 确保 `backup/` 目录下存在模板文件
- 或使用 `--no-template` 参数指定现有文件

### 问题：Python 版本过低

```
SyntaxError: ...
```

**解决方法：**
- 检查 Python 版本：`python --version`
- 如低于 3.8，请升级 Python

### 问题：缺少依赖包

```
ModuleNotFoundError: No module named 'openpyxl'
```

**解决方法：**
```bash
pip install -r requirements.txt
```
