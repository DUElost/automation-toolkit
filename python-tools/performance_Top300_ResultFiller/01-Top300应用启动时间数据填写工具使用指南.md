# Top300 Result Filler 使用指南
> **工具开发与维护者：dai.lv@tinno.com**

## 重要提示

### 首次使用必须验证数据准确性

> **警告：工具执行依赖预设的代码逻辑进行数据回填，结果不一定 100% 正确！**

首次使用时，请务必执行以下验证步骤：

1. **运行工具生成结果文件**
2. **与手动填充的数据进行逐项比对**
3. **确认数据准确性后再进行后续使用**
4. **如发现任何偏差，请立即联系工具开发者**

### 免责声明

本工具旨在提高数据填充效率，但自动化处理存在以下风险：

- 数据匹配可能因格式变化而失效
- 特殊情况可能未被代码逻辑覆盖
- 工具错误可能导致数据填入错误位置

**使用者有责任验证生成数据的准确性。如发现问题，请及时向工具开发者反馈。**

## 前置要求

### 文件路径说明

> **重要：本工具不支持直接使用网络路径（如 `\\server\share\file.xlsx`）**

如果您的测试报告文件存储在网络位置，请先将网络磁盘映射到本地驱动器：

**Windows 映射网络磁盘步骤：**

1. 打开「此电脑」或「文件资源管理器」
2. 点击顶部「映射网络驱动器」或右键「此电脑」→「映射网络驱动器」
3. 选择驱动器号（如 `Z:`）
4. 输入网络路径（如 `\\server\share`）
5. 点击「完成」

映射后，使用本地路径访问文件：
- 原路径：`\\server\share\test_report_空载.xlsx`
- 映射后：`Z:\test_report_空载.xlsx`

### Python 环境

- Python 版本：**3.8 或更高版本**
- 检查版本命令：
  ```bash
  python --version
  ```

### 安装依赖

首次使用前，需安装项目依赖：

```bash
pip install -r requirements.txt
```

依赖包列表：
- pandas >= 2.0.0
- openpyxl >= 3.1.0

## 使用流程

### 步骤 1：输入空载标准

```
请输入空载标准（直接回车默认为 /）:
```

- 输入空载条件下的标准值（如：`10`、`/`、`Mipi` 等）
- 直接回车默认使用 `/`

### 步骤 2：输入负载标准

```
请输入负载标准（直接回车默认为 /）:
```

- 输入负载条件下的标准值
- 直接回车默认使用 `/`

### 步骤 3：输入空载测试报告文件路径

```
请输入空载测试报告文件路径（如: test_report_空载.xlsx）:
```

- 输入空载测试报告 Excel 文件的完整路径
- 或文件名（如果文件在当前目录）

**示例：**
- `test_report_空载.xlsx`
- `D:\data\test_report_空载.xlsx`

### 步骤 4：输入负载测试报告文件路径

```
请输入负载测试报告文件路径（如: test_report_负载.xlsx）:
```

- 输入负载测试报告 Excel 文件的完整路径
- 或文件名（如果文件在当前目录）

### 步骤 5：输入目标Excel文件路径

```
请输入目标Excel文件路径（如: 目标文件.xlsx）:
```

- 输入目标 Excel 文件的完整路径
- 此文件不会被修改，工具会创建带时间戳的新文件

### 步骤 6：确认执行

工具显示执行摘要：

```
============================================================
请确认处理信息:
  空载标准: 10
  负载标准: /
  空载文件: test_report_空载.xlsx
  负载文件: test_report_负载.xlsx
  目标文件: 目标文件.xlsx
  输出文件: 将在原目录创建带时间戳的新文件
============================================================

确认开始处理？(Y/n):
```

确认无误后，按 `Y` 或回车开始执行。

## 执行结果

### 成功示例

```
============================================================
Top300 Result Filler - 处理结果
============================================================
✓ 处理成功

空载文件处理 (2个sheet):
  - 空载1'0078: 零值个数=5, 平均值=123.45
  - 空载2'0001: 零值个数=2, 平均值=234.56

负载文件处理 (2个sheet):
  - 负载1'0078: 零值个数=3, 平均值=145.67
  - 负载2'0001: 零值个数=1, 平均值=167.89

输出文件: D:\xxx\目标文件_20260205_103045.xlsx
============================================================
```

### 生成的文件

- **文件名格式：** `目标文件_YYYYMMDD_HHMMSS.xlsx`
- **文件位置：** `main.py` 所在目录（与 `test_report_*.xlsx` 同目录）
- **原目标文件：** 不会被修改

### 新增的 Sheet 页

工具会在输出文件中创建以下新 Sheet 页：

| Sheet 名称 | 数据来源 | 说明 |
|-----------|----------|------|
| 空载1'0078 | 空载报告第1个sheet | 复制并计算 |
| 空载2'0001 | 空载报告第2个sheet | 复制并计算 |
| 负载1'0078 | 负载报告第1个sheet | 复制并计算 |
| 负载2'0001 | 负载报告第2个sheet | 复制并计算 |

### 新增的列

每个新增的 Sheet 页会添加 `非首轮平均启动时间` 列：

| 列名 | 计算公式 |
|------|----------|
| 非首轮平均启动时间 | `=AVERAGE(Round_2:Round_8)` |

### "测试结果" Sheet 页回填

工具会自动在 "测试结果" Sheet 页中填写：

| 列 | 空载行 | 负载行 |
|------|--------|--------|
| 标准 | 用户输入的空载标准 | 用户输入的负载标准 |
| #1 | 第1个空载sheet的平均值 | 第1个负载sheet的平均值 |
| #2 | 第2个空载sheet的平均值 | 第2个负载sheet的平均值 |
| 均值 | (#1 + #2) / 2 | (#1 + #2) / 2 |

## 数据处理逻辑

### 1. Sheet 页识别

工具会自动识别包含以下表头的 Sheet 页：

```
测试APK | 时间类型 | 平均启动时间 | 最大启动时间 | 最小启动时间 | 跳变次数 | Round_1 | Round_2 | ... | Round_8
```

### 2. 数据筛选

- 筛选 `时间类型 = AM` 的数据
- 统计 `平均启动时间 = 0` 的个数（用于日志展示）

### 3. 非首轮平均值计算

- 筛选 `平均启动时间 ≠ 0` 的数据
- 计算每行的非首轮平均值：`=AVERAGE(Round_2:Round_8)`
- 计算列平均值（填入最后一行有数据下一格）

### 4. Sheet 名称映射

| 原Sheet名称 | 新Sheet名称 |
|-------------|-------------|
| `0000NX261H000078` | 空载1'0078` |
| `0000NX261G000001` | 空载2'0001` |
| `0000NX261H000078` | 负载1'0078` |
| `0000NX261G000001` | 负载2'0001` |

## 常见数据差异原因

| 原因 | 说明 |
|------|------|
| 源文件表头不匹配 | 工具依赖特定的表头格式来识别有效sheet |
| Excel 列名变化 | Round列名格式变化可能导致计算失败 |
| 数据行数不足 | 每个sheet至少需要2个有效数据用于#1/#2列回填 |
| "测试结果"sheet结构变化 | 列名或行标识变化可能导致回填失败 |

## 故障排除

### 问题：找不到源文件

```
文件不存在: test_report_空载.xlsx
```

**解决方法：**
- 检查文件路径是否正确
- 确认文件扩展名为 `.xlsx`

### 问题：未找到有效sheet

```
文件 xxx 中未找到有效sheet
```

**解决方法：**
- 确认源文件包含正确的表头
- 检查表头是否包含：`测试APK、时间类型、平均启动时间...Round_8`

### 问题：测试结果sheet缺少必要列

```
测试结果sheet缺少必要列，跳过回填。缺少列: ['标准', '#1', '#2', '均值']
```

**解决方法：**
- 确认目标文件的"测试结果"sheet包含所需的列名
- 检查列名是否完全匹配（包括符号）

### 问题：未找到空载/负载行

```
未找到'空载'行
未找到'负载'行
```

**解决方法：**
- 确认"测试结果"sheet的A列包含"空载"和"负载"文字
- 检查文字是否有前后空格

### 问题：Python 版本过低

```
SyntaxError: ...
```

**解决方法：**
- 检查 Python 版本：`python --version`
- 如低于 3.8，请升级 Python

### 问题：缺少依赖包

```
ModuleNotFoundError: No module named 'openpyxl'
```

**解决方法：**
```bash
pip install -r requirements.txt
```
