# Top300 Result Filler

> Top300应用启动时间测试数据自动处理工具
> 版本：v1.0.0 | 更新时间：2026-02-05

---

## 功能概述

自动化处理Top300应用启动时间测试数据：
1. 读取空载/负载测试报告Excel文件
2. 识别有效sheet页（按表头匹配）
3. 筛选AM时间类型数据，统计零值个数
4. 计算非首轮平均启动时间（Round_2到Round_8的平均值）
5. 在目标文件中创建新sheet页并写入数据
6. 自动回填测试结果sheet页

## 关键业务规则

### 1. Sheet页命名规则

```
原sheet名: 0000NX261H000078
新sheet名: 空载1'0078  或  负载1'0078
         ===========    ===========
         前缀+序号+分隔符+后4位
```

### 2. 数据筛选流程

```
原始数据
  ↓ 筛选 时间类型=AM
  ↓ 统计 平均启动时间=0 的个数（用于展示）
  ↓ 筛选 平均启动时间≠0
  ↓ 计算 非首轮平均启动时间 = AVERAGE(Round_2:Round_8)
  ↓ 计算 列平均值（填入最后一行）
```

### 3. 测试结果回填

```
在"测试结果"sheet中：
- 通过"标准"列识别空载/负载行
- #1列 ← 第1个sheet的平均值
- #2列 ← 第2个sheet的平均值
- 均值列 ← (#1 + #2) / 2
```

### 列名配置

```python
SOURCE_HEADERS = [
    "测试APK", "时间类型", "平均启动时间", "最大启动时间",
    "最小启动时间", "跳变次数", "Round_1", "Round_2",
    "Round_3", "Round_4", "Round_5", "Round_6", "Round_7", "Round_8"
```

### 目标文件列名

```python
COLUMN_STANDARD = "标准"
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| Excel公式在openpyxl中可能不自动计算 | 使用代码直接计算数值，确保结果一致 |
| 目标文件"测试结果"sheet结构未知 | 先探测表头结构，动态查找列索引 |
| Round_1~Round_8列位置可能变化 | 动态查找列索引，支持列顺序变化 |
| Sheet顺序与#1/#2列对应关系 | 按sheet索引顺序严格匹配，记录日志 |
